### 4.4 Transaction Scenarios

#### Scenario 1: Successful Transaction
1. Sender вызывает смарт-контракт и депонирует USDT
2. Контракт блокирует средства и создаёт transaction record
3. Обе стороны подтверждают сделку (`approve()`)
4. Средства автоматически переводятся получателю
5. Комиссия отправляется на адрес платформы

#### Scenario 2: Deadline Expiration
1. Получатель не подтверждает получение в течение 24 часов
2. Sender вызывает `claimRefundAfterDeadline()`
3. Средства возвращаются отправителю (минус комиссия)

#### Scenario 3: Dispute Resolution
1. Одна из сторон отказывается от сделки (`raiseDispute()`)
2. Арбитр получает уведомление
3. Арбитр изучает доказательства обеих сторон
4. Арбитр вызывает `resolveDispute(transactionId, decision)`
5. Средства переводятся согласно решению арбитра

### 4.5 Security Features

- **Immutability**: Контракт не может быть изменён после deployment
- **Open-Source**: Код контракта публично доступен для аудита
- **On-Chain Transparency**: Все транзакции видимы на блокчейне
- **Time-Locks**: Автоматические возвраты по истечении deadline
- **Anti-Fraud**: Механизм арбитража предотвращает мошенничество

### 7.4 Anti-Fraud Logic Summary

async function validateTransaction(userId, txid, expectedAmount) {
  // 1. Check TXID uniqueness
  if (await db.txidExists(txid)) {
    throw new Error("TXID already used");

  // 2. Verify on blockchain
  const txInfo = await tronAPI.getTransaction(txid);
  if (!txInfo || txInfo.status !== "SUCCESS") {
    throw new Error("Invalid transaction");

  // 3. Check timing (30 min window)
  const timeDiff = Date.now() - txInfo.timestamp;
  if (timeDiff > 30 * 60 * 1000) {
    throw new Error("Transaction too old");

  // 4. Verify amount
  if (txInfo.amount !== expectedAmount) {
    throw new Error("Amount mismatch");

  // 5. Save to database
  await db.recordTransaction(userId, txid, txInfo);

  return true;

- Экономия на gas fees (internal transfers)
